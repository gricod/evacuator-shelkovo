### Как скачать фото с Яндекс Карт:

1. Открыть карточку организации на Яндекс Картах
2. Открыть фото → правая кнопка → «Открыть изображение в новой вкладке»
3. В URL заменить суффикс размера на `orig` (полный размер)
4. Сохранить фото

### Как сжать фото для сайта:

```python
# Нужен Python + Pillow: pip install Pillow
from PIL import Image
import glob, os

for f in glob.glob('photos/*.jpg'):
    img = Image.open(f)
    w, h = img.size
    if w > 1200:
        img = img.resize((1200, int(h * 1200 / w)), Image.LANCZOS)
    img.save(f, 'JPEG', quality=80, optimize=True)
```

Результат: фото ~100-300 КБ вместо 1-3 МБ.

## ЭТАП 2. Структура лендинга

### 9 обязательных секций (в этом порядке):

```
1. HERO (первый экран)
   - H1 заголовок: «[Услуга] в [Город] — [УТП]»
   - 3-4 преимущества буллетами
   - Цена «от»
   - Кнопки: WhatsApp + Telegram
   - Телефон (click-to-call)
   - Блок доверия: рейтинг, режим работы

2. УСЛУГИ И ЦЕНЫ
   - 6 карточек с иконкой, названием, описанием, ценой, кнопкой

3. КАК ЭТО РАБОТАЕТ
   - 3 шага: Напишите → Подача → Доставка

4. ГАЛЕРЕЯ
   - 8 реальных фото работ (lazy loading)

5. ОТЗЫВЫ
   - 3-4 лучших отзыва с именами
   - Общий рейтинг
   - Ссылка на Яндекс Карты

6. ПРЕИМУЩЕСТВА
   - 6 блоков с иконками

7. ЗОНА ОБСЛУЖИВАНИЯ
   - Список городов с временем подачи
   - Встроенная Яндекс Карта (iframe)

8. FAQ
   - 5-7 вопросов-ответов (аккордеон)

9. CTA (финальный призыв)
   - Кнопки WhatsApp + Telegram (крупные)
   - Телефон
```

### Sticky-элементы на мобильных:
- Фиксированная панель внизу: WhatsApp | Звонок | Telegram
- Появляется после прокрутки 400px

## ЭТАП 3. SEO-оптимизация

### Meta-теги (заполнить в `<head>`):

```html
<title>[Услуга] в [Город] 24/7 — [УТП] | от [Цена] ₽</title>
<meta name="description" content="[Услуга] в [Город] круглосуточно. [Оборудование]. Подача от [X] минут. [Города]. Звоните [Телефон]">
<meta name="keywords" content="[услуга] [город], [услуга] 24 часа, [услуга] [район], [услуга] круглосуточно, [услуга] цена">
```

**Пример для эвакуатора:**
```
Title: Эвакуатор в Щёлково 24/7 — подача от 15 минут | от 2 000 ₽
Description: Эвакуация автомобилей в Щёлково круглосуточно. Сдвижная платформа, лебёдка. Подача от 15 минут. Щёлково, Балашиха, Фрязино, Москва. Звоните +7 (926) 666-23-11
```

### Schema.org разметка (JSON-LD):

Обязательно добавить два блока:
1. **LocalBusiness / ProfessionalService** — название, адрес, телефон, координаты, рейтинг, услуги, цены, зона обслуживания, часы работы
2. **FAQPage** — вопросы и ответы из секции FAQ

Шаблоны лежат в файле `site/index.html` — копировать и менять данные.

### Open Graph теги:
```html
<meta property="og:title" content="...">
<meta property="og:description" content="...">
<meta property="og:image" content="URL к фото для шаринга">
```

### Чек-лист SEO:
- [ ] Title до 60 символов с ключевым словом и городом
- [ ] Description до 160 символов с телефоном
- [ ] Keywords (Яндекс их учитывает!)
- [ ] H1 — один на страницу, с ключевым словом
- [ ] Alt-теги на всех фото с ключевиками
- [ ] Schema.org LocalBusiness + FAQPage
- [ ] Open Graph + Twitter Cards
- [ ] canonical URL
- [ ] robots.txt
- [ ] sitemap.xml
- [ ] Все изображения с loading="lazy"
- [ ] Viewport meta с viewport-fit=cover

## ЭТАП 4. Технические требования к коду

### Файловая структура:
```
project/
├── index.html          # Главная страница
├── thanks.html         # Страница «Спасибо» (noindex)
├── css/
│   └── style.css       # Все стили
├── js/
│   └── main.js         # Скрипты
├── img/                # Фото (сжатые, до 300 КБ)
├── robots.txt
├── sitemap.xml
└── CNAME               # Если свой домен
```

### CSS — ключевые правила:
- Mobile-first подход
- 5 breakpoints: 400px, 480px, 768px, 1024px, default (desktop)
- `box-sizing: border-box` на всё
- `overflow-x: hidden` на body
- `overflow-wrap: break-word` — защита от переполнения
- `-webkit-tap-highlight-color: transparent` — убрать серый квадрат на iOS
- `env(safe-area-inset-bottom)` — для iPhone с вырезом
- `scroll-behavior: smooth` + `scroll-padding-top: 80px`
- `prefers-reduced-motion` — отключать анимации для тех, кто их не хочет
- `:focus-visible` — стили фокуса для доступности

### JS — что должен делать:
- Scroll reveal анимация (Intersection Observer)
- FAQ аккордеон
- Мобильное меню (бургер)
- Sticky header при скролле
- Sticky-бар на мобильных (появляется после 400px скролла)
- Трекинг кликов по WhatsApp/Telegram/телефону (для Метрики)

### Адаптивность — что меняется на каждом breakpoint:

| Элемент | Desktop | Tablet (1024) | Mobile (768) | Small (400) |
|---------|---------|--------------|-------------|-------------|
| Услуги | 3 колонки | 2 колонки | 1 колонка | 1 колонка |
| Галерея | 4 колонки | 3 колонки | 2 колонки | 2 колонки |

## ЭТАП 6. Публикация на GitHub Pages

### Вариант А — Через браузер (для нетехнических сотрудников):

1. Войти на **github.com**
2. Создать репозиторий: **«+» → «New repository»**
   - Имя: `название-проекта`
   - Public
   - Без README
3. Нажать **«Add file» → «Upload files»**
4. Перетащить **содержимое** папки проекта (НЕ саму папку!)
   - `index.html` должен быть в корне репозитория
5. Нажать **«Commit changes»**
6. Перейти в **Settings → Pages**
7. Source: **Deploy from a branch** → **main** → **/ (root)** → **Save**
8. Через 1-2 минуты сайт доступен: `https://ЮЗЕРНЕЙМ.github.io/НАЗВАНИЕ-РЕПО/`

### Вариант Б — Через терминал:

```bash
cd путь/к/папке/проекта
git init
git add .
git commit -m "Initial release"
git branch -M main
git remote add origin https://github.com/ЮЗЕРНЕЙМ/НАЗВАНИЕ-РЕПО.git
git push -u origin main
```

### Обновление сайта:

**Через браузер:** Открыть файл на GitHub → карандаш (Edit) → внести изменения → Commit

**Через терминал:**
```bash
git add .
git commit -m "Описание изменения"
git push
```

Сайт обновляется автоматически за 1-2 минуты.

## ЭТАП 7. Подключение аналитики (после публикации)

### Яндекс Метрика:

1. Зайти на **metrika.yandex.ru**
2. «Добавить счётчик» → указать URL сайта
3. Включить: Вебвизор, Карта кликов, Карта скроллинга
4. Тип проекта: «Лидогенерация»
5. Получить 8-значный ID
6. В `index.html` раскомментировать блок Метрики, вставить ID
7. Запушить изменение

### Цели Метрики (настроить в интерфейсе):

| Цель | Тип | Условие |
|------|-----|---------|
| Клик WhatsApp | JavaScript-событие | whatsapp_click |
| Клик Telegram | JavaScript-событие | telegram_click |
| Клик Телефон | JavaScript-событие | phone_click |

Эти события уже прописаны в `main.js` — они автоматически отправляются при кликах.

### Яндекс Вебмастер:

1. Зайти на **webmaster.yandex.ru**
2. «Добавить сайт» → указать URL
3. Способ подтверждения: «Мета-тег»
4. Скопировать значение `content="..."` из предложенного тега
5. Вставить в `index.html` в строку `yandex-verification`
6. Запушить → нажать «Проверить» в Вебмастере

## ЭТАП 8. Подключение своего домена (опционально)

1. Купить домен на **reg.ru** (от 200 ₽/год)
2. В корне репозитория создать файл **CNAME** с содержимым:
   ```
   ваш-домен.ru
   ```
3. В DNS домена добавить записи:
   ```
   A     @     185.199.108.153
   A     @     185.199.109.153
   A     @     185.199.110.153
   A     @     185.199.111.153
   CNAME www   ЮЗЕРНЕЙМ.github.io
   ```
4. В Settings → Pages → Custom domain → ввести домен
5. Включить **Enforce HTTPS**
6. Обновить в `index.html`: canonical, og:url, og:image на новый домен
7. Обновить `robots.txt` и `sitemap.xml`

## Шаблон для нового проекта

Чтобы сделать лендинг для другого бизнеса:

1. Скопировать папку `site/`
2. Заменить во всех файлах:
   - Название компании
   - Телефон (во всех форматах: tel:, wa.me/, отображаемый)
   - Адрес и координаты
   - Цены и услуги
   - Отзывы
   - Фото
   - ID организации в Яндекс Картах (для iframe)
   - Ключевые слова в meta-тегах
   - Schema.org данные
3. Сжать новые фото
4. Создать новый репозиторий на GitHub
5. Загрузить файлы
6. Включить GitHub Pages

Среднее время на проект: **2-4 часа** при наличии всех данных от клиента.
